Здравствуйте. В ходе решения поставленной задачи было реализовано следующее:
- Калькулятор, который может складывать числа. Можно сложить 2 и более числа, так как я посчитал, что это подходит под требование: "В случае, если пользователь вводит что-то кроме чисел и +, в результат выводить сообщение «Error»"
  
- MVP - Model как основная логика калькулятора, View как UI элементы и Presenter, который связывает UI и Core.

- Модуль Domain - основная логика калькулятор, изолированная от внешней части проекта. Содержит: ICalculatorOperation для возможности добавления других математических операций; ICalculatorRepository для сохранение и загрузки данных; IEquationValidator для проверки введённой пользователем строки.
- Модуль диалогового окна для показа Popup в случае невалидного ввода.
- Модуль Infrastructure - Unity based UI элементы.
- Модуль Presenter - слушает и обновляет View. Для рассчётов и сохранения обращается к Model.

- Для внедрения зависимостей использовался VContainer.
- Для валидации строки использовался - Regex.
- Для сохранения состояния между сессиями использовался PlayerPrefs. Реализовано сохранение с Debounce (UniTask.Delay для задержки перед сохранением + CancellationTokenSource для отмены), так как сохранение данных только в момент закрытия приложения не всегда может отработать корректно.
- Настройки (строк, префабы, и тд.) - ScriptableObject.
- MaxHeightLayoutElement для динамического расширения истории рассчётов с фиксированой максимальной высотой.
- Для возможности складывать большие числа использовался BigInteger.
- Tools/Calculator/... для очистки PlayerPrefs.
- Validator и Calculation покрыты EditMode тестами.
- DefaultOrientation - Portrait для Android и IOS.
